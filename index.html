noesis.

Conceptos que se ven, no que se memorizan.

Un motor de animaciones explicativas. Cada idea se construye como algo que se ve ocurrir, con su texto académico al lado. No es un video, no es una diapositiva.

baja para verlo

Esto no es un video ni una diapositiva. Es una simulación que corre en tu página, generada cuadro a cuadro por el motor.

El vocabulario

Nueve maneras de hacer que una idea ocurra frente a ti.

Cada concepto pide una forma. El motor sabe contar estas nueve, y cada escena se construye deliberadamente alrededor de una.

  1. 01el viajeroAlguien recorre el cambio y lo vemos avanzar.
  2. 02la poblaciónMuchos cuerpos hacen visible lo estadístico.
  3. 03el umbralUn valor cruza una línea y algo cambia de estado.
  4. 04la convergenciaPiezas dispersas se reúnen en una.
  5. 05el antes y despuésDos mundos, un corte que los separa.
  6. 06la disecciónUna cosa se abre en sus partes con nombre.
  7. 07la analogíaUna forma se deforma hasta rimar con otra.
  8. 08la cadenaUna causa vuelca a la siguiente, en orden.
  9. 09la transformación en sitioEl orden se vuelve desorden sin moverse de lugar.

Cómo se autorea

Del concepto al JSON a la escena.
La calidad la pone quien la autoría.

Se obtiene en GitHub

noesis es open source. Lo clonas, lo sirves estático y ya tienes el motor, el repertorio y la skill de autoría. No hay paquete que instalar.

git clone https://github.com/LastrarterotH/noesis-engine
cd noesis-engine
python3 -m http.server 8765
Ver el repositorio

Se autorea con Claude Code

Las escenas se construyen dialogando el concepto con Claude Code: la skill /noesis arma el JSON con el vocabulario del motor, lo valida y lo previsualiza. Quien encarga la escena aporta el criterio, no escribe el JSON a mano.
