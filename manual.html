El manual

En preparación

Cómo se usa noesis con Claude Code.

El manual completo está en preparación. Mientras tanto, este es el camino corto para tener noesis corriendo y autorear tu primera escena.

1 · Conseguirlo

noesis es open source. Lo clonas desde GitHub, lo sirves estático y ya tienes el motor, el repertorio y la skill de autoría. No hay paquete que instalar, ni build.

git clone https://github.com/LastrarterotH/noesis-engine
cd noesis-engine
python3 -m http.server 8765

Y abres http://localhost:8765/index.html en el navegador.

2 · Autorear una escena con Claude Code

Las escenas no se escriben a mano: se construyen dialogando el concepto con Claude Code. La skill /noesis conversa la idea contigo, elige la mecánica que mejor la encarna, arma el JSON con el vocabulario del motor, lo valida y lo previsualiza. Tú aportas el criterio.

Honestamente: Claude Code tiene un costo. noesis no fuerza ser gratis al cien por ciento. Las buenas herramientas a veces se pagan.

3 · Validar y previsualizar

Cada escena se chequea contra el vocabulario real del motor antes de correr, y se prueba en un smoke test headless que la construye y la tickea para cazar errores silenciosos.

node tools/validate.mjs scenes/NN-slug.json
node tools/smoke.mjs

Luego se sirve estático y se verifica en el navegador. La verificación visual la hace el autor.

4 · Embeber una escena

Una escena terminada se incrusta en cualquier página con el Web Component, cargando el motor como módulo ES. Sin dependencias.

<script type="module" src="engine/noesis-engine.js"></script>
<noesis-scene src="scenes/01-netflix-prize.json" lang="es"></noesis-scene>
